Middleware - это функция которая позволяет обрабатывать входящие HTTP запросы еще до того как они
попадут в контролеры. Они чаще всего используются в логировании и в обработке ошибок (валидация).
Авторизацию проверяем в guards.

Guards - механизм авторизации/доступа. Они решают можно ли этому пользователю выполнить этот endpoint

Pipes - это один из ключевых механизмов валидации, трансформации и обработки данных в момент, 
когда они приходят в твой контроллер.
pipes — это функции/классы, которые могут изменить или проверить входящие данные до того, 
как они попадут в метод контроллера.

Interceptors - это класс, который позволяет:
-Обернуть ответ, чтобы любой ответ получал поле "success": true
-Замерить время выполнения
-Трансформировать данные, например убрать поле password из ответа

Основные задачи Pipes
- Валидация
Проверка данных на соответствие правилам.
Пример: проверить, что id — это UUID.
- Трансформация
Преобразование типа данных.
Пример: строка "123" → число 123.
- Очистка или подготовка данных
Удаление лишних полей, преобразование формата и т.д.

Swagger - это инструмент для документирования и тестирования REST API. Сейчас официальное название OpenAPI,
но слово Swagger до сих пор используется чаще.
Он позволяет:
- автоматически генерировать документацию API
- тестировать эндпоинты прямо в браузере
- видеть структуру запросов и ответов
- тестировать авторизацию
